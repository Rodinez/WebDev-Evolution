Anotações sobre os principais comandos do Git/Github para relembrar

    Comandos básicos sem git

  • ls - Lista todos os arquivos filhos de um arquivo pai
  • cd *nome do arquivo* - Acessa o arquivo escolhido
  • cd .. - Retorna-se à pasta anterior (arquivo pai)
  • mkdir - Cria-se uma nova pasta

    • *Sempre bom lembrar que esses comandos vão ser referentes a pasta no qual o Git Bash foi aberto*

    Comandos básicos com git

  • git config --global user.name *meu nome* - Define o nome de usuário para o Git
  • git config --global user.email *meu email* - Define o e-mail de usuário para o Git (mesmo do GitHub)
  • git init - Cria um repositório Git
  • git status - Checa o projeto e indica se existe "untracked files"
  • git add -A - Adiciona todos os arquivos que foram modificados
  • git commit -m "mensagem" - Armazena as mudanças efetuadas e a mensagem escrita
  • git push origin main - Envia todos os arquivos modificados e “commitados” para o repositório no github
  • git clone *https://github.com/user/project.git* - Copia um projeto pro seu repositório local

*Caso não tenha achado o comando que te interessa ou quer aprofundar o seu conhecimento é só clicar no logo do GitHub que o mesmo te redicionará ao glossário do GitHub*
